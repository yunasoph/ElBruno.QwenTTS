# Session Log: 1.7B Model Viability Analysis

**Date:** 2026-04-02T16:43Z  
**Agents:** Trinity (ML Engineer), Morpheus (Lead)  
**Mode:** Parallel background analysis  
**Issue:** #26 — Request for Qwen3-TTS 1.7B support

## Summary

Two specialists analyzed supporting the 1.7B model variant in parallel.

**Trinity (ML Engineer):** Technical feasibility analysis complete. 1.7B is architecturally compatible with our ONNX pipeline. Only Talker LM hidden_size changes (1024→2048); all other components identical. ONNX export feasible with config-driven changes. Resource requirements: 12-16 GB RAM, 2-2.5× slower inference. Code changes: Python exporters + C# inference need dimension-agnostic refactoring (~100+ hardcoded `1024` references).

**Morpheus (Lead):** Scope assessment complete. Medium effort (1-2 days), non-breaking change. Codebase impact: LanguageModel.cs (HIGH), EmbeddingStore.cs (MEDIUM), ModelDownloader.cs (HIGH), others not affected. Recommended phased approach: Phase 1 MVP (8-12 hours), Phase 2 optimization (future).

## Outcomes

✅ **Both recommend PURSUING** 1.7B support.
✅ **Non-breaking:** Backward compatible with 0.6B (default behavior).
✅ **High user value:** Instruction control enables emotion/rate/timbre control.
✅ **Manageable risk:** Same architecture, only configuration differs.
