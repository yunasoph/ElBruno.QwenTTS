## Key Findings

- **Architecture compatible:** Only Talker LM hidden_size changes (1024→2048); Code Predictor, vocoder, tokenizer, speakers identical
- **ONNX export feasible:** Same pipeline works with config parameterization (remove hardcoded `TALKER_HIDDEN = 1024`)

- **Resource intensive:** ~3.5 GB larger models, requires 12-16 GB RAM minimum, ~2-2.5× slower inference
- **Code changes needed:** Python exporters + C# inference hardcode dimension `1024` in ~100+ locations

## Effort Estimate

- C# refactor: 4-6 hours (LanguageModel buffer allocation, EmbeddingStore dimension-awareness)
