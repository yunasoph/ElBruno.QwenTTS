## Decisions Recorded

### Algorithm Selection: Min-Heap Over Full Sort
- **Why**: O(n log k) vs O(n log n) — 3× speedup for k=10, n=1000
- **Design**: Internal MinHeap class with bubble-up/bubble-down, only accepts values > minimum when full
- **SIMD**: TensorPrimitives.Dot() for cosine similarity, TensorPrimitives.Norm() for L2 normalization
- **Streaming**: Heap enables online/streaming search (references don't need to fit in memory at once)

### Normalization Strategy
- Automatic L2 normalization of query and reference embeddings
- Zero vector edge case: Copy as-is if norm < 1e-8f
- Cosine similarity remains in [-1, 1] range

### EmbeddingStore Integration
- New public method: `GetSpeakerEmbedding(int speakerId)` — retrieves single speaker embedding (1024-dim)
- New public method: `GetAllSpeakerEmbeddings()` — yields (name, embedding) tuples for all speakers
- Optimized for Top-K iteration without copying entire matrix

## Performance Characteristics
- **Baseline**: 7.11 ms average for Top-10 from 1000 speakers (1024-dim, 100 iterations)
- **Theoretical**: 1000 speakers, k=10: min-heap 3,322 ops vs full sort 9,966 ops
- **Speedup**: ~3× expected improvement
